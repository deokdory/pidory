# pidory v0.7.0 — First Public Release

## English

### Highlights
- **First public release** of pidory under Apache-2.0 license
- **Database migration**: SQLite → PostgreSQL (breaking change)
- **Multi-user support** via Discord sender prefix
- **/update slash command** with pre-flight validation and pg_dump backup
- **i18n** with Korean (해요체) and English locales

### Breaking Changes
- **PostgreSQL required** (#304, #307): pidory now requires PostgreSQL 14+. SQLite is no longer supported.
  - Set `DATABASE_URL` environment variable
  - Run `sudo bash scripts/postgres-setup.sh` for automatic role/database/db.env setup
  - The `[database] path` field in `config.toml` is deprecated and ignored
- **License changed**: MIT → Apache-2.0. See [LICENSE](../../LICENSE) for the full text.

### New Features
- Discord sender prefix for multi-user threads (#316, #324) — distinguishes speakers in shared sessions
- /update command with pre-flight validation (#338) — blocks update if PostgreSQL setup is incomplete
- /update DB backup·restore via pg_dump/psql (#339) — automatic backup before migration
- claude_settings atomic editor (#295) — 7-layer concurrency protection for permission persistence
- 3-row permission button layout (#297, #298) — improved Discord UI for Allow/AlwaysAllow/Deny
- Tool kind cache (#306) — same-turn tool re-use auto-passes
- MCP tool permission exact-form rule (#310) — fixes infinite prompt loop
- Permission persistence with 5 protected prefixes (#318) — safer Always Allow scope
- /clear archived tombstone fix (#321)
- i18n cleanup with consistent 해요체 tone (#341)
- v0.7.0 misc bundle (#340): Fixes #288, #290, #293, #313

### Migration Guide
1. **Backup existing SQLite database** (if upgrading from v0.6.x):
   ```bash
   cp pidory.db pidory.db.backup
   ```
2. **Install PostgreSQL** and run setup:
   ```bash
   sudo bash scripts/postgres-setup.sh
   ```
   This creates the `pidory` role/database and writes `/etc/pidory/db.env` (mode 600).
3. **Update via /update slash command** (recommended) or rebuild manually:
   ```bash
   git pull && cargo build --release
   sudo systemctl restart pidory
   ```
4. **Verify**:
   ```bash
   systemctl status pidory
   journalctl -u pidory -f
   ```
